# بازطراحی نمودارهای داشبورد و نمای پروژه‌ها

## هدف

در این تغییر، نمودارهای تکراری بین صفحات حذف شدند و هر صفحه فقط اطلاعات مناسب همان سطح مدیریتی را نمایش می‌دهد.

## تقسیم مسئولیت صفحات

### `/dashboard`

داشبورد اصلی برای عملیات روزانه است و فقط این موارد را نشان می‌دهد:

- شاخص‌های فوری سامانه
- روند ۷ روز اخیر گزارش‌های کاری و وظایف تکمیل‌شده
- آخرین رویدادهای سامانه
- دسترسی سریع به پروژه‌ها، تقویم و نمای کلان پروژه‌ها

در این صفحه نمودارهای وضعیت پروژه، اولویت پروژه و وضعیت وظایف حذف شدند تا با صفحه نمای کلان تکراری نباشند.

### `/dashboard/project-overview`

نمای کلان پروژه‌ها برای تحلیل پورتفو است و این موارد را نمایش می‌دهد:

- نقشه سلامت پروژه‌ها با Treemap
- ترکیب وضعیت پروژه‌ها با Pie chart
- اولویت پروژه‌ها با Bar chart
- وضعیت وظایف کل پروژه‌ها با Bar chart
- ریسک عقب‌افتادگی بر اساس نقش
- ریسک عقب‌افتادگی بر اساس کارشناس
- حجم گزارش‌ها بر اساس پروژه
- جدول پروژه‌های عقب‌افتاده
- جدول بار کاری نقش‌ها و کارشناسان

### `/dashboard/projects/[id]`

صفحه جزئیات هر پروژه نمودارهای اختصاصی همان پروژه را نمایش می‌دهد:

- وضعیت وظایف پروژه
- اولویت وظایف پروژه
- مدت فازهای پروژه
- روند پیشرفت گزارش‌شده
- دسته‌بندی فایل‌های همان پروژه

## تغییرات UI/UX

- همه صفحات راست‌چین هستند.
- خود نمودارها با `dir="ltr"` رندر می‌شوند تا محورهای Recharts به‌درستی نمایش داده شوند.
- کارت‌های نمودار ساده‌تر و خواناتر شدند.
- فازهای پروژه در صفحه جزئیات به‌صورت ردیف کامل نمایش داده می‌شوند و اطلاعات اضافی مثل افراد فاز در این بخش حذف شده است.
- صفحه داشبورد اصلی دیگر نقش صفحه تحلیل پورتفو را ندارد.
