# Shamsi Date Picker

## هدف

تمام فیلدهای انتخاب تاریخ در فرانت‌اند با کامپوننت مشترک `ShamsiDateInput` مدیریت می‌شوند. کاربر تاریخ را با تقویم شمسی انتخاب می‌کند، اما مقدار ذخیره‌شده و ارسال‌شده به API همچنان به‌صورت میلادی `YYYY-MM-DD` باقی می‌ماند.

## کامپوننت اصلی

مسیر:

```text
src/components/common/ShamsiDateInput.tsx
```

این کامپوننت از پکیج‌های زیر استفاده می‌کند:

```text
react-multi-date-picker
react-date-object/calendars/persian
react-date-object/locales/persian_fa
```

## رفتار داده

- مقدار ورودی از فرم یا API به‌صورت میلادی دریافت می‌شود.
- در UI با تقویم شمسی و زبان فارسی نمایش داده می‌شود.
- بعد از انتخاب تاریخ، خروجی دوباره به `YYYY-MM-DD` میلادی تبدیل می‌شود.
- این کار باعث می‌شود backend بدون تغییر باقی بماند و فقط تجربه کاربری تاریخ‌ها شمسی شود.

## صفحات و بخش‌های تغییر یافته

- فرم تعریف پروژه
- فرم ایجاد پروژه
- فرم ویرایش پروژه
- فرم اعضای پروژه
- فرم ایجاد و ویرایش وظیفه
- صفحه جزئیات پروژه

## وابستگی لازم

چون آرشیو ارسالی فقط شامل `src` بود، فایل `package.json` در دسترس نبود. قبل از اجرای پروژه باید این پکیج نصب شود:

```bash
npm i react-multi-date-picker
```
