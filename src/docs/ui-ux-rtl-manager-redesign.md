# بازطراحی تجربه کاربری مدیریتی و نمودارهای RTL

## هدف

این تغییرات برای این انجام شد که مدیر بتواند وضعیت پروژه را در چند ثانیه تشخیص دهد و صفحات شلوغ به محیط‌های کاری مرحله‌ای و قابل اسکن تبدیل شوند.

## اصول طراحی اعمال‌شده

- استفاده از DaisyUI برای تب‌ها، Alert، Stat، Badge، Button، Input و فرم‌ها.
- نمایش اطلاعات تصمیم‌ساز در خلاصه مدیریتی، نه پخش‌کردن همه اطلاعات در یک صفحه.
- جداسازی اطلاعات پروژه در تب‌های مستقل تا فقط یک بخش فعال نمایش داده شود.
- بهینه‌سازی نمودارهای فارسی/راست‌به‌چپ با محور، Tooltip و Legend خواناتر.
- جلوگیری از خروج متن از خانه‌های نقشه سلامت پروژه‌ها.

## تغییرات اصلی

### صفحه جزئیات پروژه

- ناوبری Anchor حذف و با تب‌های DaisyUI جایگزین شد.
- تب‌های جدید:
  - خلاصه مدیریتی
  - فازها
  - اعضا
  - وظایف
  - گزارش کار
  - فایل‌ها
  - نمودارها
- در حالت عادی فقط تب فعال نمایش داده می‌شود.
- بخش خلاصه مدیریتی شامل وضعیت فوری، درصد تکمیل وظایف، روزهای باقی‌مانده، وظایف عقب‌افتاده و جمع مالی ساده فازهاست.

### نمودارها

- نمودار اولویت وظایف در جزئیات پروژه دیگر متن بیرونی روی PieChart ندارد؛ راهنما در پایین نمودار آمده تا متن فارسی بیرون نزند.
- محور نمودار مدت فازها برای RTL بهتر روی سمت راست قرار گرفت.
- متن‌های Treemap نقشه سلامت پروژه‌ها داخل ClipPath قرار گرفتند تا از باکس‌ها خارج نشوند.

### هدف مدیریتی

مدیر باید در نگاه اول این موارد را ببیند:

- پروژه قابل کنترل است یا نیاز به اقدام فوری دارد.
- چند وظیفه تکمیل شده و چند وظیفه عقب‌افتاده است.
- چند روز تا سررسید مانده یا چند روز از سررسید گذشته است.
- وضعیت مالی ساده فازها از نظر درآمد واقعی، هزینه واقعی و مانده واقعی چیست.
