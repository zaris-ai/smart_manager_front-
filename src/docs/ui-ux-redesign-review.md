# بازطراحی ظاهری صفحات داشبورد

## هدف

این تغییرات برای خلوت‌سازی صفحات داشبورد، کاهش تکرار نمودارها، یکپارچه‌سازی کارت‌ها و بهتر شدن خوانایی صفحات مدیریتی انجام شده است.

## الگوی ظاهری جدید

الگوی اصلی رابط کاربری به شکل داشبورد مدیریتی آرام‌تر پیاده‌سازی شده است:

- هدر یکپارچه برای صفحات اصلی
- کارت‌های آماری محدود و قابل اسکن
- نوار فیلتر فشرده
- ردیف‌های کامل برای لیست پروژه‌ها
- منوی سه‌نقطه برای عملیات ردیفی
- کارت‌های بخش‌بندی‌شده با `base-*` و سازگار با تم روشن/تاریک
- حذف فرم‌های همیشه‌نمایان در صفحه نقش‌ها
- جمع‌شدن نمودارهای سنگین در صفحه جزئیات پروژه

## کامپوننت‌های مشترک اضافه‌شده

فایل جدید:

```text
src/components/common/DashboardUi.tsx
```

کامپوننت‌های اصلی:

- `DashboardPageHeader`: هدر مشترک صفحات مدیریتی
- `AdminStatCard`: کارت آماری خلوت و استاندارد
- `SectionCard`: کارت بخش‌بندی‌شده برای فرم، جدول و نمودار
- `FilterBar`: نوار فیلتر فشرده
- `SoftBadge`: بج نرم‌تر برای وضعیت‌ها و هشدارها

این کامپوننت‌ها از `src/components/common/index.ts` نیز export شده‌اند.

## صفحات تغییر یافته

### داشبورد اصلی

فایل:

```text
src/components/dashboard/DashboardTestCharts.tsx
```

تغییرات:

- تعداد کارت‌های آماری از حالت شلوغ به ۴ شاخص اصلی کاهش یافت.
- هدر صفحه ساده‌تر شد.
- تمرکز صفحه روی عملیات روزانه، فعالیت‌های اخیر و روند ۷ روزه باقی ماند.

### لیست پروژه‌ها

فایل:

```text
src/pages/dashboard/projects/index.tsx
```

تغییرات:

- کارت‌های سنگین پروژه به ردیف‌های کامل تبدیل شدند.
- اطلاعات اضافی مثل لیست کامل اعضا از لیست حذف شد.
- عملیات مشاهده، ویرایش و حذف به منوی سه‌نقطه منتقل شد.
- حذف پروژه با `confirmToast` انجام می‌شود و `window.confirm` حذف شد.

### تعریف پروژه

فایل‌ها:

```text
src/pages/dashboard/projects/define.tsx
src/pages/dashboard/projects/create.tsx
```

تغییرات:

- فرم به بخش‌های واضح‌تر تقسیم شد.
- فازها به کارت‌های جمع‌شونده تبدیل شدند.
- بخش مالی ساده هر فاز حفظ شد: درآمد پیش‌بینی‌شده و هزینه پیش‌بینی‌شده.
- خلاصه ثبت پروژه در ستون کناری باقی ماند.

### جزئیات پروژه

فایل:

```text
src/pages/dashboard/projects/[id].tsx
```

تغییرات:

- هدر پروژه خلوت‌تر و استانداردتر شد.
- ناوبری سریع بین فازها، اعضا، وظایف، گزارش‌ها و فایل‌ها اضافه شد.
- نمودارهای اختصاصی پروژه داخل بخش جمع‌شونده قرار گرفتند تا صفحه شلوغ نشود.
- کارت‌های اصلی به ظاهر `base-*` نزدیک‌تر شدند.

### تقویم پروژه‌ها

فایل:

```text
src/pages/dashboard/calendar/index.tsx
```

تغییرات:

- فیلترها به نوار فشرده منتقل شدند.
- تقویم داخل `SectionCard` قرار گرفت تا تمرکز صفحه روی نمای تقویم باشد.
- هدر صفحه با الگوی جدید هماهنگ شد.

### کاربران

فایل:

```text
src/pages/dashboard/users/index.tsx
```

تغییرات:

- تعداد کارت‌های آماری به ۳ کارت مهم کاهش یافت.
- جدول کاربران خواناتر شد.
- عملیات کاربر به منوی سه‌نقطه منتقل شد.
- راهنما به‌صورت پیش‌فرض بسته است.
- غیرفعال‌سازی کاربر با `confirmToast` انجام می‌شود.
- `Toaster` تکراری از صفحه حذف شد و فقط `Toaster` سراسری `_app.tsx` باقی ماند.

### نقش‌های پروژه

فایل‌ها:

```text
src/components/roles/ProjectRolesPage.tsx
src/components/projects/roles.tsx
```

تغییرات:

- فرم ایجاد/ویرایش نقش از حالت همیشه‌نمایان به مودال تبدیل شد.
- لیست نقش‌ها اولویت اصلی صفحه شد.
- عملیات نقش‌ها داخل منوی سه‌نقطه قرار گرفت.
- مسیر قدیمی نقش‌ها به wrapper سازگار با صفحه جدید تبدیل شد.
- `window.confirm` قدیمی حذف شد.

### گزارش‌های مدیریتی و نمای کلان

فایل‌ها:

```text
src/components/project-overview/ProjectOverviewPage.tsx
src/components/project-charts/ProjectChartsPage.tsx
```

تغییرات:

- هدرهای گرادیانی سنگین با کارت ساده‌تر جایگزین شدند.
- کلاس‌های ظاهری به `base-*` نزدیک‌تر شدند تا با تم کلی هماهنگ‌تر باشند.

## تصمیم مهم طراحی

این تغییرات عمداً همه اطلاعات را از صفحات حذف نکرده‌اند. هدف اصلی این بود که اطلاعات ثانویه از نمای اولیه عقب‌تر برود و صفحات به مسئولیت اصلی خود برگردند:

- داشبورد: عملیات روزانه
- پروژه‌ها: لیست سریع و قابل اسکن
- جزئیات پروژه: فضای کاری پروژه
- نمودارها: گزارش مدیریتی
- تقویم: نمای زمان‌بندی
- کاربران: مدیریت حساب‌ها
- نقش‌ها: تنظیمات نقش‌های پروژه
